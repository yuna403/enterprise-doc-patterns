创建订单 - 组合模式 (Composite)

SimpleOrder 实现 OrderComponent 接口，是组合模式的叶子节点。
CompositeOrder 可包含多个子订单，总金额为子订单金额之和，是组合模式的容器节点。

审批订单 - 责任链模式 (Chain of Responsibility)

ManagerApprover 处理金额 < 1000 的订单。
DirectorApprover 处理金额 < 5000 的订单。
审批链：经理 → 总监，逐级审批，超出权限则传递到下一级。

生成文档 - 组合模式 + 装饰器模式 (Decorator)

CompositeElement 包含多个 TextElement，组成文档树（组合模式）。
BoldDecorator 包装文档组件，添加 <b> 标签格式化输出（装饰器模式）。

保存文件 - 策略模式 (Strategy)

HtmlSaveStrategy 实现 SaveStrategy 接口，将文档保存为 HTML 和 TXT 两种格式。
通过策略模式，可以轻松切换不同的保存方式（如 PDF、Word 等）。

遍历文件 - 迭代器模式 + 访问者模式

FileSystemIterator 使用栈实现深度优先遍历（迭代器模式）。
StatsVisitor 收集每个节点的统计信息（访问者模式）。
Directory + File 构成树形文件系统（组合模式）。

支付订单 - 策略模式 (Strategy)

CreditCardStrategy 信用卡支付策略。
AlipayStrategy 支付宝支付策略。
通过策略模式，运行时可动态切换支付方式，无需修改代码。
